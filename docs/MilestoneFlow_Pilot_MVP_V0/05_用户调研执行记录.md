## 2. 调研目标

验证七个核心问题：

1. 范围失控是否高频并造成真实时间或金额损失。
2. 当前客户确认是否依赖聊天、口头或不可追溯方式。
3. 阶段交付是否需要具体版本级验收。
4. 验收延迟是否影响尾款。
5. 用户是否愿意在额外工作前使用正式变更单。
6. 客户是否愿意通过免注册网页链接完成确认。
7. 用户是否愿意投入真实项目、订金或付费。

## 3. 样本目标

### 深度访谈：20 人

| 行业 | 目标人数 |
|---|---:|
| 软件、网站、小程序开发 | 8 |
| UI、平面、品牌设计 | 5 |
| 视频、摄影、内容制作 | 3 |
| 营销代运营 | 2 |
| 独立顾问 | 2 |

团队规模建议：独立工作者 8 人、2～5 人工作室 8 人、6～10 人工作室 4 人。

### 定量问卷

- 最低 50 份有效问卷；
- 建议目标 100 份；
- 无项目经验、重复填写、明显机器人和关键题缺失不计入有效样本。

### 原型测试

- 第一轮 5 人，先发现 S3/S4 阻断问题；
- 修复后第二轮 5 人；
- 至少 2 人使用手机测试客户公开页；
- 至少 2 人愿意提供真实项目。

### 真实项目 Pilot

- 招募 5～10 名种子用户；
- 至少 3 个用户愿意使用真实项目数据；
- 至少 2 个用户愿意付费、支付可退订金或给出明确付款承诺，作为进入完整开发的重要门槛。

## 4. 样本筛选

必须满足：

- 最近 12 个月完成至少 2 个付费客户项目；
- 至少 1 个固定价或阶段付款项目；
- 本人参与报价、交付、验收或催款中的至少两项；
- 能描述最近一次真实事件，而不是只讨论想象场景。

排除：

- 从未做过商业客户项目；
- 只做全职内部项目；
- 完全按小时且没有交付验收；
- 无法描述真实流程；
- 仅因奖励报名且不符合目标画像。

## 6. 小红书招募文案

### 标题

**最近一次客户说“只是一个小修改”，你最后加价了吗？**

### 正文

我们正在研究固定价客户项目中的真实问题：

- 项目范围只散落在聊天记录；
- 客户中途临时加需求；
- 阶段成果交付后迟迟不验收；
- 尾款因为验收或争议一直卡住；
- 报价、交付、变更和回款分散在多个工具。

现招募有真实接案经验的独立开发者、小型软件/设计/视频工作室和顾问，参加一次约 40～50 分钟的线上访谈。

适合参与：

- 过去 12 个月至少做过 2 个付费客户项目；
- 做过固定价或阶段付款项目；
- 参与过报价、交付、验收或催款。

这不是销售，不会公开你的姓名、客户名称或敏感项目内容。访谈前会说明记录和录音方式，你可以跳过任何问题。

报名时请私信：

1. 你的工作类型；
2. 过去一年大约做过多少个客户项目；
3. 是否遇过客户临时加需求或尾款延迟；
4. 最近一周方便访谈的时间。

## 11. 反馈分类与进入 Backlog 规则

每条反馈记录：

| 字段 | 说明 |
|---|---|
| 反馈编号 | FB-来源-序号 |
| 来源 | 访谈/问卷/原型/Pilot/缺陷 |
| 用户原话或行为 | 不使用产品团队改写后的措辞替代 |
| 标签 | 痛点、行为、需求、采用、付费 |
| 频率 | 同类证据出现次数 |
| 严重度 | 1～5 |
| 当前方案不满意度 | 1～5 |
| 付费相关度 | 1～5 |
| 核心闭环相关性 | 必要/增强/无关 |
| 结论 | V0.1 CR / V0.2 / Later / Reject |

机会分：

```text
发生频率 × 严重度 × 当前方案不满意度 × 付费相关度
```

用户提出功能不等于进入 V0.1。默认进入 V0.2，由证据评审。

## 12. 每 5 次访谈的复盘

必须输出：

1. 已验证和被推翻的假设；
2. 高频真实事件和损失；
3. 不同行业差异；
4. 客户免注册链接采用阻力；
5. 正式变更单使用阻力；
6. 真实项目与付费信号；
7. V0.2 新增项；
8. 是否达到 Go/Pivot/Stop 条件。

## 13. 隐私与知情同意

- 录音前必须获得明确同意。
- 不记录客户名称、完整合同、账号、令牌或商业机密。
- 用户提供材料需脱敏，未经许可不得发送给 AI。
- 参与者可跳过问题、终止访谈或要求删除资料。
- 研究结果引用采用匿名编号。
- 原始数据、整理数据和分析数据分层保存，不覆盖原始记录。
