## 2. 基线目的

将原始 MVP V1.0 从“完整可销售产品”裁剪为一个能验证核心差异化和真实付费信号的 Pilot 版本。V0.1 不通过功能数量竞争，而验证以下假设：

1. 固定价项目服务者愿意结构化记录工作范围与非工作范围。
2. 客户愿意通过免注册安全链接确认报价、交付和需求变更。
3. 不可变交付版本和版本级验收可以减少版本混乱与验收争议。
4. 正式需求变更可以在额外工作开始前明确费用和工期影响。
5. 验收、变更和应收关联能改善尾款跟踪与催款。
6. 用户愿意将真实项目放入系统，并产生继续使用或付费信号。

## 3. 产品定位

> 面向固定价项目服务者的项目范围、交付验收与回款控制平台。

首批用户聚焦：

- 1～5 人软件、网站、小程序开发团队；
- 独立开发者；
- UI、品牌和设计工作室；
- 视频、内容制作团队；
- 独立顾问。

## 4. Pilot 角色

| 角色 | V0.1 权限 |
|---|---|
| Owner | 登录管理端，管理本工作空间全部客户、项目、报价、交付、变更、应收和审计 |
| Client | 无需注册，通过单对象安全链接查看并确认/拒绝报价、验收交付、确认/拒绝变更 |
| System Job | 计算逾期、处理邮件任务和重试，不拥有交互式账号 |

**裁剪说明：** Admin、Member、Viewer、成员邀请和多工作空间切换进入 V0.2。Pilot 每个工作空间仅允许一个内部 Owner，避免权限矩阵掩盖核心价值验证。

## 5. 核心业务闭环

```text
注册并创建工作空间
→ 创建客户和项目
→ 编辑并发布报价版本
→ 客户免注册确认报价
→ 生成当前商业基线与应收草稿
→ 创建里程碑并提交交付版本
→ 客户通过/请求修改/拒绝
→ 验收通过激活应收
→ 超范围时创建正式需求变更
→ 客户确认后更新商业基线和应收
→ 记录部分/全部付款
→ 查看待付款和已逾期并发送提醒
```

## 7. 详细验收基线

### PRD-PILOT-AUTH-001 账号注册与会话

**需求说明：** 支持邮箱注册、邮箱验证、登录、刷新、退出和密码重置；发送公开业务对象前邮箱必须已验证。  
**原始来源：** PRD 11.1；17  
**SRS 映射：** FR-AUTH-001～008  
**页面：** PAGE-P01

**功能级验收：**

- 合法邮箱和符合策略的密码可以注册，重复邮箱和弱密码被明确拒绝。
- 邮箱验证令牌一次有效且可过期；未验证账号不得发送客户链接。
- 登录成功签发会话，退出、重置密码或账号禁用后旧会话失效。
- 连续失败登录受到限制，错误信息不暴露账号是否存在。

### PRD-PILOT-WS-001 单工作空间启动与租户边界

**需求说明：** 注册后创建一个工作空间并设置名称、默认币种、时区和联系邮箱；Pilot 内部仅开放 Owner 操作，多成员管理进入 V0.2。  
**原始来源：** PRD 11.2；14；Pilot 裁剪决策  
**SRS 映射：** FR-WS-001、002、007；DR-003  
**页面：** PAGE-P02

**功能级验收：**

- 首次用户可创建工作空间并自动成为 Owner。
- 金额使用工作空间默认币种，业务时间按工作空间时区计算和展示。
- 任何业务查询和写入均校验 workspace_id，跨工作空间访问被拒绝。
- Pilot 不提供成员邀请、角色调整和多工作空间切换入口。

### PRD-PILOT-DASH-001 商业事项工作台

**需求说明：** 突出待报价确认、待客户验收、待变更确认、待付款、已逾期和下一动作，不建设通用任务看板。  
**原始来源：** PRD 8.1；12.1；原型 WF-01  
**SRS 映射：** FR-PROJ-005、FR-REC-012  
**页面：** PAGE-P03

**功能级验收：**

- 用户在一个页面看到影响收入的待办和金额汇总。
- 统计值只包含当前工作空间、未归档业务数据。
- 点击卡片可进入对应项目、验收、变更或应收。
- 无数据时显示建立首个客户项目的引导。

### PRD-PILOT-CLIENT-001 客户与联系人最小管理

**需求说明：** 创建、编辑、查询和归档客户，保留一个主要联系人；标签、复杂历史统计和 CSV 导入进入 V0.2。  
**原始来源：** PRD 11.3  
**SRS 映射：** FR-CLIENT-001～005、007  
**页面：** PAGE-P04

**功能级验收：**

- 可创建个人或公司客户并维护主要联系人邮箱。
- 可按名称、联系人和邮箱搜索。
- 存在项目、报价或应收的客户不得物理删除。
- 归档客户不可创建新项目，但历史数据仍可查看。

### PRD-PILOT-PROJ-001 项目创建、状态与归档

**需求说明：** 为活跃客户创建项目，维护项目名称、日期、币种和商业状态；归档后只读。  
**原始来源：** PRD 11.4  
**SRS 映射：** FR-PROJ-001～007  
**页面：** PAGE-P05、PAGE-P06

**功能级验收：**

- 只能为活跃客户创建项目。
- 项目状态转换必须符合 Pilot 状态机，不能任意跳转。
- 项目详情显示总额、已收、待收、下一里程碑与待处理事项。
- 归档项目不得新增报价、交付、变更、应收或付款记录。

### PRD-PILOT-BASE-001 当前商业基线

**需求说明：** 以已确认报价加已确认变更计算当前有效范围、非范围、交付物、验收标准、金额、付款计划和交付日期。  
**原始来源：** PRD 11.5、11.7、12.3；原型 5.1  
**SRS 映射：** FR-QUOTE-014、FR-CR-005、007、010；DR-005  
**页面：** PAGE-P06

**功能级验收：**

- 未确认报价或变更不得影响商业基线。
- 确认变更后生成新的基线快照并保留变更前历史。
- 项目总金额与当前交付日期由有效基线计算，不由前端自行拼接。
- 用户可追溯每项范围和金额来自哪个报价或变更版本。

### PRD-PILOT-QUOTE-001 报价草稿与发布校验

**需求说明：** 编辑工作范围、非工作范围、交付物、验收标准、修改次数、金额、付款计划、有效期和交付日期。  
**原始来源：** PRD 11.5.1；原型 WF-04  
**SRS 映射：** FR-QUOTE-001～003  
**页面：** PAGE-P07

**功能级验收：**

- 草稿可反复保存但不会生成公开链接或改变项目基线。
- 发布前至少有一项工作范围、合法总金额、币种、有效期和正确的付款计划合计。
- 工作范围与非工作范围必须分别展示。
- 校验失败时保留输入并定位错误字段。

### PRD-PILOT-QUOTE-002 不可变报价版本与发送

**需求说明：** 发布后创建递增、不可变报价版本和单对象公开链接；支持发送邮件、复制链接、撤销和创建新版本。  
**原始来源：** PRD 11.5.2～11.5.3；原型 5.2、WF-04  
**SRS 映射：** FR-QUOTE-004～008、012、013、015；FR-PUBLIC-001～005、009  
**页面：** PAGE-P07、PAGE-P16

**功能级验收：**

- 每次发布生成不可变版本和递增版本号。
- 同一项目同一时刻只有一个有效待确认报价版本。
- 新版本发布后旧待确认链接不可再操作。
- 邮件失败不回滚发布结果，用户仍可复制公开链接。

### PRD-PILOT-QUOTE-003 客户免注册确认报价

**需求说明：** 客户通过安全链接查看报价版本，确认、拒绝或留言；确认后建立首个商业基线和应收草稿。  
**原始来源：** PRD 11.5.4～11.5.5；12.5；原型 WF-05  
**SRS 映射：** FR-QUOTE-008～014；FR-PUBLIC-002～009  
**页面：** PAGE-P08、PAGE-P16

**功能级验收：**

- 客户无需注册，只能访问链接绑定的单一报价版本。
- 确认前必须明确展示版本、范围、非范围、金额、付款计划、日期和确认后果。
- 确认操作幂等；重复请求不产生重复基线、应收或审计记录。
- 过期、撤销、无效或被替代链接不泄露项目和客户信息。

### PRD-PILOT-MS-001 里程碑管理

**需求说明：** 在已确认项目中创建、排序、启动和取消里程碑，设置计划日期、金额/比例与验收标准。  
**原始来源：** PRD 11.6.1～11.6.2  
**SRS 映射：** FR-MS-001～003  
**页面：** PAGE-P06、PAGE-P09

**功能级验收：**

- 里程碑顺序保存后保持稳定。
- 里程碑验收标准可从报价基线继承并允许草稿期补充。
- 已进入待验收或已验收的里程碑不得物理删除。
- 状态只能按 NOT_STARTED→IN_PROGRESS→PENDING_ACCEPTANCE 等合法路径转换。

### PRD-PILOT-DEL-001 不可变交付版本

**需求说明：** 提交交付说明、文件和外部链接，生成递增且不可变的交付版本，并发送验收链接。  
**原始来源：** PRD 11.6.3；原型 5.3、WF-06  
**SRS 映射：** FR-MS-004～006、012；FR-FILE-001～007  
**页面：** PAGE-P09、PAGE-P16

**功能级验收：**

- 交付草稿可编辑，提交后内容和文件引用不可覆盖。
- 再次交付创建新版本并保留旧版本及其客户反馈。
- 未完成上传或无权限文件不能进入正式交付。
- 提交成功后里程碑进入待验收并创建客户验收链接。

### PRD-PILOT-ACC-001 客户对具体交付版本验收

**需求说明：** 客户通过免注册链接对当前有效交付版本执行通过、请求修改或拒绝；反馈与版本绑定。  
**原始来源：** PRD 11.6.4；12.6；原型 WF-07  
**SRS 映射：** FR-MS-007～011；FR-PUBLIC-007～009  
**页面：** PAGE-P10、PAGE-P16

**功能级验收：**

- 客户清楚看到版本、交付说明、文件、验收标准和三种结论。
- 请求修改和拒绝必须填写具体原因。
- 已验收版本不可再次改变结论；被新版本替代的链接只读。
- 通过验收后更新里程碑并激活对应应收，整个处理必须幂等。

### PRD-PILOT-CR-001 需求变更草稿与发布

**需求说明：** 对照当前商业基线记录新增、修改、删除、仍不包含内容，以及费用、工期、交付日期和付款影响。  
**原始来源：** PRD 11.7.1～11.7.3；原型 WF-08  
**SRS 映射：** FR-CR-001～004、007、009  
**页面：** PAGE-P11

**功能级验收：**

- 变更必须关联当前项目和有效报价/基线。
- 必须同时表达范围差异与费用/工期影响；无影响时也需明确为零。
- 发布后形成不可变版本和客户链接，确认前不改变商业基线。
- 变更可撤销或过期，历史仍可追溯。

### PRD-PILOT-CR-002 客户确认需求变更

**需求说明：** 客户免注册查看变更依据和商业影响，确认、拒绝或留言；确认后更新商业基线和应收。  
**原始来源：** PRD 11.7.2～11.7.3；原型 WF-09  
**SRS 映射：** FR-CR-004～010；FR-PUBLIC-001～009  
**页面：** PAGE-P12、PAGE-P16

**功能级验收：**

- 页面明确展示新增费用、工期、新日期、付款变化和仍不包含内容。
- 确认后才生成新商业基线；拒绝后原基线保持不变。
- 重复确认不重复增加项目金额、应收或时间线记录。
- 无效、撤销、过期和被替代链接进入统一安全错误页。

### PRD-PILOT-REC-001 应收生成、状态与逾期

**需求说明：** 从已确认报价付款计划、已验收里程碑或已确认变更创建/激活应收，计算待付款、部分付款、已付款和已逾期。  
**原始来源：** PRD 11.8.1～11.8.2；原型 WF-10  
**SRS 映射：** FR-REC-001～003、005～010、012～013  
**页面：** PAGE-P03、PAGE-P06、PAGE-P13

**功能级验收：**

- 应收来源可追溯到报价付款节点、里程碑验收或变更。
- 金额使用 Decimal，已收、未收和状态由后端计算。
- 到期日结束后未付清的应收展示逾期标识。
- 界面明确说明应收和付款记录不代表平台处理真实资金。

### PRD-PILOT-PAY-001 手工记录部分或全部付款

**需求说明：** 记录付款金额、日期、方式、参考号和备注，重新计算应收状态；错误记录使用作废而非无痕删除。  
**原始来源：** PRD 11.8.3；Pilot 审计增强  
**SRS 映射：** FR-REC-004～008、011  
**页面：** PAGE-P13

**功能级验收：**

- 付款金额必须大于零并使用应收币种。
- 部分付款、全额付款和超额付款按照规则计算；超额付款需要二次确认。
- 作废错误记录需要原因并保留审计，不直接物理删除。
- 重复请求不得创建重复付款记录。

### PRD-PILOT-REM-001 到期/逾期识别与人工发送提醒

**需求说明：** 工作台识别即将到期和已逾期应收，用户预览并手工发送提醒；自动周期催款规则进入 V0.2。  
**原始来源：** PRD 11.9；原型 17.5；Pilot 裁剪决策  
**SRS 映射：** FR-NOTIF-002～005、007、009；FR-REC-009  
**页面：** PAGE-P03、PAGE-P13、PAGE-P14

**功能级验收：**

- 系统准确展示即将到期与已逾期应收。
- 用户发送前可预览项目、金额、到期日和提醒文案。
- 同一应收同一提醒动作使用幂等机制，避免重复邮件。
- 发送失败可重试并允许复制必要信息继续人工沟通。

### PRD-PILOT-NOTIF-001 关键业务邮件任务

**需求说明：** 报价、交付验收、需求变更和应收提醒通过异步邮件任务发送；站内通知中心进入 V0.2。  
**原始来源：** PRD 11.9；Pilot 裁剪决策  
**SRS 映射：** FR-NOTIF-002～005、009  
**页面：** PAGE-P07、PAGE-P09、PAGE-P11、PAGE-P14

**功能级验收：**

- 业务事务提交与邮件发送解耦，邮件失败不丢失核心业务记录。
- 系统记录邮件待发送、成功和失败状态。
- 临时失败按配置重试，永久失败不无限重试。
- 任何失败页面均提供复制公开链接作为备用。

### PRD-PILOT-FILE-001 交付文件安全管理

**需求说明：** Pilot 只开放交付文件上传、授权下载和历史引用保护；报价附件、付款凭证等扩展进入 V0.2。  
**原始来源：** PRD 11.6.3；Pilot 裁剪决策  
**SRS 映射：** FR-FILE-001～007  
**页面：** PAGE-P09、PAGE-P10

**功能级验收：**

- 限制文件类型、大小和数量，并校验扩展名、MIME 和大小。
- 下载必须校验内部工作空间权限或公开链接对象权限。
- 私有文件使用短时访问地址或受控转发。
- 已被历史交付版本引用的文件不得物理删除。

### PRD-PILOT-AUDIT-001 项目时间线与不可变审计

**需求说明：** 记录报价、公开访问、确认、交付、验收、变更、应收、付款、提醒和高风险操作。  
**原始来源：** PRD 11.11；原型 10.5  
**SRS 映射：** FR-AUDIT-001～005  
**页面：** PAGE-P06、PAGE-P15

**功能级验收：**

- 关键状态变化与对应业务写操作保持事务一致。
- 记录操作者/来源、动作、对象、摘要、服务器时间和 request_id。
- 普通业务接口不能修改或删除审计记录。
- 日志不记录密码、完整令牌、密钥或不必要的敏感全文。

### PRD-PILOT-PUBLIC-001 单对象公开链接安全

**需求说明：** 公开链接使用高熵令牌、数据库仅存哈希，支持有效期、撤销、访问记录、速率限制、信息最小化和幂等。  
**原始来源：** PRD 13.1、14；产品原则 5；SRS 3.13  
**SRS 映射：** FR-PUBLIC-001～009；NFR-SEC-*  
**页面：** PAGE-P08、PAGE-P10、PAGE-P12、PAGE-P16

**功能级验收：**

- 数据库不保存可直接使用的完整公开令牌。
- 链接只访问一个已发布对象，不可横向访问其他项目或版本。
- 过期、撤销、错误和猜测令牌返回统一最小信息。
- 确认、验收和变更操作具备速率限制与幂等保护。

### PRD-PILOT-ERR-001 一致的异常、冲突与恢复

**需求说明：** 定义字段校验、未认证、无权限、状态冲突、重复操作、网络失败、邮件失败和未知错误的用户反馈。  
**原始来源：** PRD 13；16；SRS 9  
**SRS 映射：** API-002、API-004；NFR-REL-002～005  
**页面：** 全部页面

**功能级验收：**

- 字段错误显示在对应输入附近，并保留用户已输入内容。
- 状态冲突和重复操作返回稳定错误码或同一幂等结果。
- 未知错误显示 request_id，不暴露堆栈、SQL 或敏感配置。
- 业务成功但邮件失败时，界面明确区分业务结果和通知结果。

## 9. 状态模型

### 9.1 项目状态

```text
DRAFT
→ PENDING_QUOTE_CONFIRMATION
→ CONFIRMED
→ IN_PROGRESS
→ PENDING_ACCEPTANCE
→ COMPLETED
→ ARCHIVED
```

补充分支：`PAUSED`、`CANCELLED`。归档后只读。

### 9.2 报价状态

```text
DRAFT → PUBLISHED/SENT → VIEWED
                         ├→ CONFIRMED
                         ├→ REJECTED
                         ├→ EXPIRED
                         └→ REVOKED
```

发布新版本后旧待确认版本为 `SUPERSEDED`，只读不可操作。

### 9.3 里程碑状态

```text
NOT_STARTED → IN_PROGRESS → PENDING_ACCEPTANCE
                              ├→ ACCEPTED
                              ├→ REVISION_REQUIRED → PENDING_ACCEPTANCE
                              └→ CANCELLED
```

拒绝验收作为验收结论记录，里程碑进入 `REVISION_REQUIRED` 并标记严重拒绝原因，避免新增无法闭合的状态。

### 9.4 变更状态

```text
DRAFT → SENT → VIEWED
               ├→ CONFIRMED
               ├→ REJECTED
               ├→ EXPIRED
               └→ REVOKED
```

### 9.5 应收状态

```text
DRAFT → PENDING → PARTIALLY_PAID → PAID
                    └────────────→ OVERDUE（以 overdue_flag 叠加）
```

Pilot 固定采用：主状态保留 `PARTIALLY_PAID`，到期未付清时增加 `overdue=true`，避免丢失付款进度。

## 10. Pilot 明确边界

### 10.1 V0.1 不做

- AI 范围、变更和催款草案；
- 内部多成员协作和复杂角色；
- 通用任务、工时、甘特图、日程和即时聊天；
- 站内通知中心；
- 自动连续催款；
- 在线支付、资金托管、税务发票；
- 法定电子签章或合同法律审查；
- 原生移动 App；
- 多语言、多币种换算；
- 自定义模板、品牌域名、高级报表；
- 开放 API、Webhook 和第三方集成。

### 10.2 固定产品原则

1. 客户无需注册。
2. 公开链接只访问一个业务对象。
3. 已发布报价、交付和变更版本不可覆盖。
4. 未确认变更不改变商业基线。
5. 平台不处理真实资金，付款记录由用户手工登记。
6. 金额使用 Decimal，时间以工作空间时区解释、数据库存 UTC。
7. 业务成功不得因邮件失败而回滚。
8. 所有客户确认动作必须幂等并进入审计。
9. 公开确认不宣称法定电子签章。
10. Pilot 仅允许一个内部 Owner。

## 12. 非功能基线

- 管理端面向桌面浏览器；客户公开页移动端优先。
- 除文件上传和异步邮件外，核心 API P95 目标小于 500ms；公开确认同步结果小于 2 秒。
- Pilot 目标可用性 99.5%，计划维护除外。
- 关键写操作使用数据库事务，确认、验收、变更、付款使用幂等机制。
- 租户隔离、公开链接、文件授权和日志脱敏必须通过安全测试。
- 生产环境必须具备 HTTPS、备份、恢复验证、健康检查和错误追踪。
- 网络失败后尽量保留表单输入；错误响应包含稳定 code 与 requestId。

## 13. 发布门禁

1. 22 项 P0 需求全部映射到 SRS、页面、API 和测试。
2. 报价确认、交付验收、变更确认、应收付款四条 E2E 主流程通过。
3. 所有公开链接的过期、撤销、被替代、猜测和重复提交测试通过。
4. 租户隔离、文件越权、金额计算和状态机测试通过。
5. 业务成功但邮件失败时可复制链接继续流程。
6. 无未关闭 P0/P1 阻断缺陷。
7. 重复请求导致的重复基线、应收、付款或审计记录为 0。
8. 数据库迁移、备份、恢复和回滚方案验证通过。
9. 隐私政策、服务条款、公开确认免责声明和基础帮助页可访问。
10. 项目总控窗口已登记 V0.1 冻结清单和版本标签。

## 14. 需求变更规则

- 新需求进入 `07_V0.2_Backlog与变更管理.md`，不直接改 V0.1。
- 可申请 V0.1 变更的原因仅限：安全、合规、数据丢失、核心流程阻断、验收标准冲突。
- 变更请求需记录 CR 编号、来源、证据、影响的 PRD/SRS/页面/API/测试/数据和决策。
- 批准后的修订版本采用 V0.1.x，不复用或重排需求编号。
