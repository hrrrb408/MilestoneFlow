# 《MilestoneFlow V0.2 Backlog 与 V0.1 变更管理》

## 1. 原则

- V0.1 已冻结需求不得被新建议直接覆盖。
- 所有新增、增强和非阻断体验建议进入 V0.2 Backlog。
- 用户提出次数不是唯一优先级依据，必须结合真实事件、损失、采用阻力和付费信号。
- V0.1 只接受安全、合规、数据丢失、核心流程阻断和验收冲突类 CR。
- V0.2 候选可以被拒绝，不代表承诺开发。

## 4. 评审规则

### 可进入 V0.1.x

- 导致公开链接越权或数据泄露；
- 导致金额、基线、应收或付款计算错误；
- 导致客户确认后数据未生效或重复生效；
- 导致真实项目核心闭环无法完成；
- PRD 与 SRS 验收条件互相冲突；
- 违反已明确的隐私或合规要求。

### 必须进入 V0.2

- 新增角色、页面、报表、模板、AI 或第三方集成；
- 优化视觉但不阻断任务；
- 增加通用项目管理能力；
- 自动化程度提升；
- 单个用户偏好且缺少重复证据；
- 需要改变产品定位或商业模式。

## 5. Backlog 优先级建议

```text
证据强度 × 痛点严重度 × 出现频率 × 付费相关度 × 核心闭环相关性
────────────────────────────────────────────────────────────
实现成本 × 安全/合规风险 × 对现有基线扰动
```

评审时必须保留反证：不需要、现有工具已足够、客户拒绝、输入太重、无付费意愿。
