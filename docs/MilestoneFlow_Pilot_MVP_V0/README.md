## 本次校正

上一版临时基线因未读取原始文件，将产品错误收敛为“项目进度与客户审核工具”。本校正版恢复原始产品的真正差异化闭环：

```text
创建客户与项目
→ 定义工作范围与非工作范围
→ 发布不可变报价版本
→ 客户免注册确认
→ 形成当前商业基线
→ 提交不可变交付版本
→ 客户验收/请求修改/拒绝
→ 创建并确认正式需求变更
→ 更新商业基线
→ 激活应收、记录付款与跟踪逾期
```

## Pilot 裁剪结论

### V0.1 保留

- 单 Owner 工作空间与租户隔离；
- 客户和项目最小管理；
- 当前商业基线；
- 报价范围、不可变版本和免注册确认；
- 里程碑、交付版本和客户验收；
- 正式需求变更及费用/工期影响；
- 应收、部分付款、逾期和人工提醒；
- 交付文件；
- 邮件任务、公开链接、审计与幂等安全。

### 移入 V0.2

- AI 草案；
- 成员邀请和复杂角色；
- 多工作空间；
- 站内通知中心；
- 自动周期催款；
- 自定义模板、导出、品牌化、报表；
- 支付、电子签章、API、移动 App、第三方集成。

## 冻结规则

- 新功能不得直接写入 V0.1，统一登记至 V0.2 Backlog。
- 只有安全、合规、数据丢失、核心流程阻断或验收歧义可提出 V0.1 变更请求。
- 已发布需求编号不得复用、重排或删除；废弃时标记 Deprecated。
- 所有变更必须同步影响的 PRD、SRS、页面、API、测试和成功指标。
