# 《MilestoneFlow Pilot 成功指标定义》

## 1. 指标目的

Pilot 用于判断是否继续投入完整产品，而不是证明代码已经完成。成功必须同时包含：

- 问题真实；
- 客户外部流程可完成；
- 用户愿意使用真实项目；
- 核心商业闭环发生；
- 出现付费或强承诺；
- 系统足够安全、稳定和可追溯。

## 2. 核心定义

### 2.1 有效目标用户

最近 12 个月至少完成 2 个付费客户项目，且参与报价、交付、验收或催款中的至少两项。

### 2.2 有效 Pilot 团队

满足：

- 非项目内部测试账号；
- 有真实或高度真实的客户项目；
- 创建客户、项目和报价；
- 至少向一名真实客户发送公开链接。

### 2.3 激活

注册后 7 天内完成：

```text
创建工作空间
→ 创建客户和项目
→ 发布第一份报价
```

### 2.4 核心价值闭环

至少完成：

```text
客户确认报价
→ 提交一个交付版本
→ 客户通过验收或请求修改
```

加强闭环还包括任一：

- 客户确认正式需求变更；
- 验收或变更激活应收；
- 记录部分或全部付款；
- 发送一次真实到期/逾期提醒。

### 2.5 强付费信号

从强到弱：

1. 已付款；
2. 支付可退订金；
3. 接受早鸟报价并明确付款日期；
4. 提供真实项目并让客户参与；
5. 只表示“有兴趣”不计强付费信号。

## 3. 北极星指标

> **完成“客户确认报价 + 版本级交付验收”的真实 Pilot 项目数。**

该指标同时验证服务方输入意愿、客户打开链接意愿、版本验收价值和核心流程可靠性。

## 4. 阶段门禁

### 4.1 研究验证门禁

| 指标 | 目标 |
|---|---:|
| 深度访谈 | 20 人 |
| 有效问卷 | ≥50，建议 100 |
| 原型测试 | 10 人 |
| 范围失控近 6 个月发生率 | ≥60% |
| 可描述时间或金额损失 | ≥50% |
| 主要依赖聊天/口头确认 | ≥50% |
| 有阶段交付且验收不清 | ≥50% |
| 曾因验收延迟回款 | ≥40% |
| 表示愿意真实使用正式变更 | ≥40% |
| 愿意在真实项目试用 | ≥30% |
| 愿意立即付费或订金 | ≥10% |

### 4.2 进入完整开发前的最低门禁

| 指标 | 最低要求 |
|---|---:|
| 愿意试用的目标用户 | ≥5 |
| 愿意使用真实项目数据 | ≥3 |
| 付费、订金或明确付款承诺 | ≥2 |
| 用户正确理解产品不是通用项目管理/开票工具 | 多数核心样本 |
| 客户愿意使用免注册链接 | 原型完成率 ≥80% |

未达到时继续原型或人工服务试点，不扩大开发范围。

### 4.3 V0.1 真实 Pilot Go 门槛

沿用项目立项目标：

| 编号 | 指标 | Go 门槛 |
|---|---|---:|
| M-PILOT-01 | 目标用户访谈 | ≥20 |
| M-PILOT-02 | 创建真实项目的用户 | ≥10 |
| M-PILOT-03 | 完成客户确认或验收的项目 | ≥5 |
| M-PILOT-04 | 付费用户或等值早鸟订单 | ≥3 |
| M-PILOT-05 | 第 4 周仍发生核心业务动作的试用用户 | ≥30% |
| M-PILOT-06 | 确认报价+交付验收完整闭环 | ≥3 个真实项目 |
| M-PILOT-07 | 已确认需求变更 | ≥1 个真实项目，或证据说明样本期无真实变更事件 |
| M-PILOT-08 | 客户公开链接完成率 | ≥80% |
| M-PILOT-09 | 未关闭 P0 阻断缺陷 | 0 |
| M-PILOT-10 | 越权、数据泄露或错误商业结果 | 0 |

| 交付验收完成率 | ≥85% |

| 理解“确认前不生效” | ≥80% |
| 理解交付版本 | ≥80% |

## 7. 首次价值时间

| 指标 | 目标 |
|---|---:|
| 注册到发布第一份报价 | ≤10 分钟（有项目资料前提） |
| 客户打开到完成报价确认 | ≤3 分钟 |
| Owner 找到最高优先级商业事项 | ≤30 秒 |

## 9. Go / Iterate / Pivot / Stop

### Go

- V0.1 真实 Pilot 的关键门槛达标；
- 至少 3 个项目完成完整闭环；
- 至少 3 个付费或早鸟订单；
- 无安全阻断；
- 用户明确将范围、验收或变更视为核心价值。

### Iterate

- 有真实闭环和付费信号，但客户链接完成率、首次价值时间或留存不足；
- 问题主要是文案、步骤、默认值或 onboarding；
- 不扩大功能，先修核心漏斗。

### Pivot

- 痛点成立，但用户真正愿意付费的是合同、支付或其他相邻问题；
- 客户普遍拒绝网页链接；
- 正式变更使用阻力不可克服；
- 软件/设计之外的某个行业信号明显更强。

### Stop

- 无法找到 5 名真实试用者或 3 个真实项目；
- 核心痛点在目标样本中频率和损失均低；
- 需要大量人工代操作才能完成客户流程；
- 无法满足公开链接、租户或文件安全要求；
- 无可验证付费信号。
