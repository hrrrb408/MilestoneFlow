# 《MilestoneFlow Pilot MVP V0.1 需求追踪矩阵》

## 1. 追踪规则

```text
原始 PRD 章节
→ Pilot PRD 稳定编号
→ 原始 SRS FR/NFR/DR 编号
→ 用户故事
→ 页面
→ API
→ 测试用例
```

- Pilot 编号用于冻结和变更管理。
- SRS 编号沿用 MF-SRS-001，不重新编号。
- 一个 Pilot 需求可映射多个 SRS 项。
- API 路径为 V0.1 契约候选，需在接口与联调窗口冻结 OpenAPI 后更新为 `Approved`。
- 测试编号为质量窗口的用例命名基线。

## 3. 覆盖检查

- [ ] 每个 P0 需求至少有一个用户故事。
- [ ] 每个 P0 需求至少映射一个原 SRS 编号。
- [ ] 每个页面动作都有 API 或明确标记纯展示。
- [ ] 每个 API 有正常、校验、权限、状态冲突和幂等测试。
- [ ] 所有公开链接测试过无效、过期、撤销、被替代和暴力猜测。
- [ ] 所有金额路径测试部分付款、全额、超额、作废和逾期。
- [ ] 所有版本对象测试不可变和历史可追溯。
- [ ] V0.2 项目未混入 V0.1 页面、API 或测试范围。

## 4. 重点 E2E

| E2E 编号 | 业务链路 |
|---|---|
| E2E-PILOT-001 | 注册→工作空间→客户→项目→发布报价 |
| E2E-PILOT-002 | 客户打开报价→确认→商业基线→应收草稿 |
| E2E-PILOT-003 | 里程碑→交付 V1→请求修改→交付 V2→通过→激活应收 |
| E2E-PILOT-004 | 当前基线→创建变更→客户确认→新基线→新增应收 |
| E2E-PILOT-005 | 应收→部分付款→逾期→提醒→全额付款 |
| E2E-PILOT-006 | 邮件失败→复制公开链接→客户成功操作 |
| E2E-PILOT-007 | 报价/交付/变更链接撤销、过期、被替代和重复操作 |
| E2E-PILOT-008 | 跨工作空间与跨公开对象访问全部失败 |
