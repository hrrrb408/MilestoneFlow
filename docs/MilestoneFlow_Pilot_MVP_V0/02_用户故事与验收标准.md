# 《MilestoneFlow Pilot MVP V0.1 用户故事与验收标准》

## 1. 文档规则

- 用户故事编号：`US-领域-序号`。
- 每个 P0 需求至少对应一个用户故事。
- 验收标准采用 Given / When / Then。
- “完成”不仅要求页面可操作，还要求权限、状态、数据、审计和幂等结果正确。
- 客户公开端必须同时在桌面与手机宽度测试。
- 所有时间使用工作空间时区展示，服务器时间用于审计。

## 2. 横向验收规则

1. 所有内部 API 校验当前工作空间。
2. 所有公开 API 只授权一个业务对象和必要文件。
3. 所有金额使用 Decimal，禁止 float/double。
4. 所有发布版本不可覆盖。
5. 所有确认动作使用幂等键或等效唯一约束。
6. 所有高风险动作写入审计。
7. 邮件失败不得回滚报价、交付或变更发布。
8. 未确认变更不得改变商业基线。
9. 记录付款不等于平台收款。
10. 未关闭 P0/P1 阻断缺陷时不得进入真实项目 Pilot。
