# 《MilestoneFlow Pilot MVP V0.1 数据库表结构》

## 1. 数据库基线

- 数据库：PostgreSQL 17.x。
- 默认 Schema：`public`；当模块数量增长后可评估按模块 Schema 拆分，但 V0.1 不增加迁移复杂度。
- 主键：`uuid`，由应用生成。
- 金额：`numeric(19,4)`。
- 技术时间：`timestamptz`，统一 UTC 写入。
- 业务日期：`date`。
- 枚举：`varchar` + CHECK + Java Enum，不使用 PostgreSQL 原生 ENUM。
- JSON：`jsonb` 仅承载设置、不可变快照、脱敏审计元数据和幂等响应，不替代核心关系字段。

## 2. 命名规范

| 对象 | 规范 | 示例 |
|---|---|---|
| 表 | 小写 snake_case，单数或明确记录名 | `workspace_membership`、`payment_record` |
| 主键 | `id` | `id uuid` |
| 外键 | `<entity>_id` | `project_id` |
| 时间 | `*_at` 技术时间，`*_date` 业务日期 | `created_at`、`due_date` |
| 布尔 | `is_*`，但优先用状态枚举表达 | `is_resolved` |
| 唯一约束 | `uk_<table>_<columns>` | `uk_project_workspace_no` |
| 外键 | `fk_<child>_<parent>` | `fk_milestone_project_workspace` |
| CHECK | `ck_<table>_<rule>` | `ck_receivable_amounts` |
| 索引 | `idx_<table>_<purpose>` | `idx_project_workspace_status_updated` |

## 5. 代表性 DDL 草案

```sql
CREATE TABLE project (
  id uuid PRIMARY KEY,
  workspace_id uuid NOT NULL REFERENCES workspace(id),
  client_id uuid NULL,
  current_baseline_id uuid NULL,
  project_no varchar(40) NOT NULL,
  name varchar(180) NOT NULL,
  description text NULL,
  currency char(3) NOT NULL,
  status varchar(40) NOT NULL DEFAULT 'DRAFT',
  start_date date NULL,
  target_end_date date NULL,
  confirmed_amount numeric(19,4) NOT NULL DEFAULT 0,
  archived_at timestamptz NULL,
  archived_by uuid NULL,
  version bigint NOT NULL DEFAULT 0,
  created_at timestamptz NOT NULL DEFAULT now(),
  created_by uuid NULL,
  updated_at timestamptz NOT NULL DEFAULT now(),
  updated_by uuid NULL,
  CONSTRAINT uk_project_workspace_id UNIQUE (workspace_id, id),
  CONSTRAINT uk_project_workspace_no UNIQUE (workspace_id, project_no),
  CONSTRAINT ck_project_currency CHECK (currency ~ '^[A-Z]{3}$'),
  CONSTRAINT ck_project_amount CHECK (confirmed_amount >= 0),
  CONSTRAINT ck_project_dates CHECK (
    start_date IS NULL OR target_end_date IS NULL OR start_date <= target_end_date
  )
);

ALTER TABLE project
  ADD CONSTRAINT fk_project_client_workspace
  FOREIGN KEY (workspace_id, client_id)
  REFERENCES client (workspace_id, id);
```

```sql
CREATE TABLE action_item_projection (
  id uuid PRIMARY KEY,
  workspace_id uuid NOT NULL REFERENCES workspace(id),
  project_id uuid NULL,
  source_type varchar(40) NOT NULL,
  source_id uuid NOT NULL,
  action_type varchar(48) NOT NULL,
  status varchar(24) NOT NULL DEFAULT 'OPEN',
  priority varchar(16) NOT NULL DEFAULT 'MEDIUM',
  title varchar(200) NOT NULL,
  summary varchar(500),
  due_at timestamptz,
  amount_impact numeric(19,4),
  currency char(3),
  deep_link varchar(500),
  source_event_id uuid,
  first_detected_at timestamptz NOT NULL DEFAULT now(),
  last_refreshed_at timestamptz NOT NULL DEFAULT now(),
  closed_at timestamptz,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now(),
  CONSTRAINT fk_action_project_workspace
    FOREIGN KEY (workspace_id, project_id)
    REFERENCES project (workspace_id, id),
  CONSTRAINT ck_action_amount_currency CHECK (
    (amount_impact IS NULL AND currency IS NULL)
    OR (amount_impact IS NOT NULL AND currency IS NOT NULL)
  )
);

CREATE UNIQUE INDEX uk_action_open_source
  ON action_item_projection(workspace_id, source_type, source_id, action_type)
  WHERE status = 'OPEN';
```

## 6. 不可变表保护

正式实现应对 `quote_version`、`delivery_version`、`change_version`、`commercial_baseline`、`audit_event` 建立数据库保护。建议将“正文快照”与“生命周期状态”分开，或通过触发器仅允许白名单状态字段变化，禁止修改内容、金额、版本号和来源。

```sql
CREATE FUNCTION reject_immutable_update() RETURNS trigger AS $$
BEGIN
  RAISE EXCEPTION 'IMMUTABLE_RECORD';
END;
$$ LANGUAGE plpgsql;
```

实际触发器须按表设计允许必要的 `REVOKED/SUPERSEDED` 状态记录更新，不能粗暴阻止合法撤销流程。

## 7. JPA Schema 管理

- `spring.jpa.hibernate.ddl-auto=validate`。
- 所有结构变更由 Flyway 管理。
- CI 在空库和“上一版本数据库”上执行迁移并启动应用验证映射。
- 禁止 Hibernate 自动创建或更新生产表。
