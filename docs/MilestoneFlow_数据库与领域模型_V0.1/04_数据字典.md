# 《MilestoneFlow Pilot MVP V0.1 数据字典》

## 1. 通用字段字典

| 字段 | 类型 | 语义 | 规则 |
|---|---|---|---|
| id | uuid | 记录主键 | 应用生成，不暴露自增序号 |
| workspace_id | uuid | 租户归属 | 除全局身份表外业务表必填 |
| project_id | uuid | 项目归属 | 必须与 workspace_id 组合引用 Project |
| version | bigint | 乐观锁版本 | 从 0 开始，每次更新递增 |
| created_at | timestamptz | 创建时间 | UTC，默认 now() |
| created_by | uuid | 创建者 User | System/Client 动作可空，Actor 写审计事件 |
| updated_at | timestamptz | 更新时间 | UTC |
| updated_by | uuid | 更新者 User | 可空 |
| archived_at | timestamptz | 归档时间 | 归档时必填 |
| archived_by | uuid | 归档操作者 | 与 archived_at 配对 |
| status | varchar | 生命周期状态 | Java Enum + DB CHECK |
| currency | char(3) | ISO 4217 币种 | 大写三字母 |
| amount_* | numeric(19,4) | 金额 | 禁止 float/double |
| *_date | date | 业务日期 | 按 Workspace 时区解释 |
| *_at | timestamptz | 技术时间点 | UTC |
| metadata / snapshot | jsonb | 白名单元数据或不可变快照 | 不替代核心 FK/金额/状态 |

## 2. 核心实体字段字典

### 2.1 User / `app_user`

| 字段 | 长度/精度 | 业务含义 | 校验/敏感级别 |
|---|---:|---|---|
| email | 320 | 展示邮箱 | PII；日志脱敏 |
| email_normalized | 320 | 唯一性和登录查询值 | PII；全局唯一 |
| display_name | 100 | 管理端显示名 | 可展示 |
| password_hash | 255 | 密码编码结果 | Secret；禁止审计/日志 |
| status | 32 | 账号生命周期 | 枚举 |
| locale | 16 | UI/邮件语言 | BCP 47 风格字符串 |
| email_verified_at |  | 邮箱完成验证时间 | UTC |
| last_login_at |  | 最近成功登录 | 安全审计 |

### 2.2 Workspace / `workspace`

| 字段 | 长度/精度 | 业务含义 | 校验 |
|---|---:|---|---|
| name | 120 | 工作空间名称 | 非空、trim 后至少 1 字符 |
| slug | 80 | URL/展示标识 | 小写字母、数字、连字符；全局唯一 |
| default_currency | 3 | 新项目默认币种 | ISO 4217 |
| timezone | 64 | 业务日期解释时区 | IANA Zone ID |
| status | 24 | ACTIVE/SUSPENDED/ARCHIVED | 枚举 |
| settings | jsonb | 链接有效期、提醒冷却等低风险配置 | JSON Schema/应用校验 |

### 2.3 WorkspaceMember / `workspace_membership`

| 字段 | 业务含义 | 校验 |
|---|---|---|
| workspace_id | 所属租户 | FK Workspace |
| user_id | 内部账号 | FK User |
| role | 工作空间角色 | V0.1 仅 OWNER 可用 |
| status | 成员状态 | PENDING/ACTIVE/REMOVED |
| joined_at | 生效时间 | ACTIVE 时必填 |
| ended_at | 终止时间 | REMOVED 时必填 |

### 2.4 Project / `project`

| 字段 | 长度/精度 | 业务含义 | 校验 |
|---|---:|---|---|
| client_id | UUID | 委托客户 | 同工作空间，可空 |
| current_baseline_id | UUID | 当前有效商业基线 | 同项目，可空 |
| project_no | 40 | 工作空间内项目编号 | `(workspace_id, project_no)` 唯一 |
| name | 180 | 项目名称 | 非空 |
| description | text | 内部项目描述 | 不进入普通日志 |
| currency | 3 | 项目结算币种 | 创建后受控不可变 |
| status | 40 | 项目生命周期 | 状态机控制 |
| start_date | date | 计划/实际开始日期 | 不晚于 target_end_date |
| target_end_date | date | 当前目标交付日 | 可由确认变更更新投影 |
| confirmed_amount | 19,4 | 当前基线确认金额 | >= 0，投影值 |

### 2.5 Milestone / `milestone`

| 字段 | 长度/精度 | 业务含义 | 校验 |
|---|---:|---|---|
| sequence_no | integer | 项目内排序 | >= 1，项目内唯一 |
| code | 40 | 可读编号 | 项目内可选唯一 |
| title | 180 | 里程碑标题 | 非空 |
| description | text | 交付描述 | 可空 |
| acceptance_criteria | text | 客户验收标准 | 提交交付前建议必填 |
| planned_start_date | date | 计划开始日 | 可空 |
| due_date | date | 业务到期日 | 按 Workspace 时区判断逾期 |
| status | 40 | 里程碑状态 | 状态机控制 |
| current_delivery_version_id | uuid | 当前提交版本 | 同里程碑 |
| accepted_at | timestamptz | 验收通过时间 | ACCEPTED 时必填 |

### 2.6 Task / `action_item_projection`

| 字段 | 业务含义 | 校验 |
|---|---|---|
| source_type | 来源对象类型 | 枚举 |
| source_id | 来源对象 ID | 与 source_type 配套 |
| action_type | 建议下一动作 | 枚举 |
| status | OPEN/DONE/DISMISSED/SUPERSEDED | 投影状态 |
| priority | LOW/MEDIUM/HIGH/CRITICAL | 枚举 |
| title | 面向 Owner 的短标题 | 不包含敏感全文 |
| summary | 脱敏摘要 | 最长 500 |
| due_at | 技术到期时间 | 从业务日期转换时使用 Workspace Zone |
| amount_impact | 商业金额影响 | 有值时必须同时有 currency |
| deep_link | 管理端内部路由 | 禁止外部 URL 注入 |
| source_event_id | 生成投影的事件 | 用于幂等和重建 |

### 2.7 ActivityLog / `audit_event`

| 字段 | 业务含义 | 校验/安全 |
|---|---|---|
| aggregate_type/id | 被操作聚合 | 必填 |
| event_type | 业务动作代码 | 稳定、不复用语义 |
| actor_type | USER/CLIENT/SYSTEM/JOB | 必填 |
| actor_user_id | 内部操作者 | actor_type=USER 时必填 |
| actor_public_link_id | 外部来源链接 | actor_type=CLIENT 时建议必填 |
| request_id | API 请求 ID | 可空但在线请求应提供 |
| source | API/PUBLIC/WORKER/MIGRATION | 必填 |
| summary | 可读摘要 | 不写敏感全文 |
| metadata | 白名单扩展数据 | 禁止 Token、Cookie、密码、预签名 URL |
| ip_hash / user_agent_hash | 安全追踪摘要 | 原文不落库或按策略短期保留 |
| occurred_at | 事件发生时间 | UTC，不可修改 |

### 2.8 Feedback / `feedback_projection`

| 字段 | 业务含义 | 校验 |
|---|---|---|
| source_type | 原始反馈记录类型 | QUOTE_DECISION/ACCEPTANCE_DECISION/CHANGE_DECISION |
| source_id | 原始反馈记录 ID | 同源唯一 |
| source_version_id | 被反馈的发布版本 | 必填 |
| feedback_type | 反馈语义 | 枚举 |
| content | 反馈正文 | 敏感业务文本；不写日志 |
| author_type | CLIENT/USER/SYSTEM | 枚举 |
| author_name_snapshot | 当时展示名 | 可空，避免依赖后续改名 |
| source_status | 原始决定状态 | 例如 REJECTED/REVISION_REQUESTED |
| is_resolved | 是否已由后续业务动作解决 | 投影值，不修改原始记录 |
| occurred_at | 原反馈时间 | UTC |

## 3. 重要支持字段字典

### 3.1 发布版本

| 字段 | 适用表 | 说明 |
|---|---|---|
| version_no | quote_version/delivery_version/change_version/baseline | 父对象内从 1 递增 |
| snapshot | 发布版本 | 发布时冻结的 JSONB 快照，结构有 schemaVersion |
| published_at / submitted_at | 发布版本 | 对外正式生效时间 |
| status | 发布版本 | PUBLISHED/CONFIRMED/REJECTED/REVOKED/SUPERSEDED 等 |
| source_*_version_id | baseline | 商业事实来源链 |

### 3.2 Receivable 与 Payment

| 字段 | 说明 | 规则 |
|---|---|---|
| source_type/source_id/source_sequence | 应收生成来源 | 同来源序号唯一 |
| amount_due | 应收总额 | >= 0 |
| amount_paid | 有效付款合计 | >= 0；由事务维护 |
| outstanding_amount | 不建议落库；查询表达式 | `amount_due - amount_paid` |
| due_date | 到期业务日 | Workspace Zone 判断逾期 |
| reference_no | 外部付款参考号 | 工作空间内可选唯一 |
| void_reason | 作废原因 | VOIDED 时必填 |

### 3.3 PublicLink

| 字段 | 说明 | 安全规则 |
|---|---|---|
| token_hash | 原始能力令牌 SHA-256 | 全局唯一；原 Token 不落库 |
| object_type/object_id | 单一授权对象 | 不允许会话切换对象 |
| allowed_actions | 允许动作集合 | 建议字符串数组/bitset，后端白名单解析 |
| expires_at | 到期时间 | 必填 |
| revoked_at/superseded_at | 失效事实 | 与 status 一致 |
| first_viewed_at/last_viewed_at/access_count | 访问统计 | 不作为授权唯一来源 |

### 3.4 IdempotencyRecord

| 字段 | 说明 | 规则 |
|---|---|---|
| scope | 业务作用域 | 内部为 Workspace，公开为 PublicLink 派生范围 |
| operation | 操作名 | 稳定枚举/代码 |
| idempotency_key | 客户端高熵 Key | 唯一范围的一部分 |
| request_hash | 规范化请求 SHA-256 | 同 Key 不同 Hash 返回冲突 |
| status | PROCESSING/COMPLETED/FAILED | 处理中可返回 Retry-After |
| response_snapshot | 首次响应白名单快照 | JSONB，不含 Cookie/Token |
| expires_at | 清理时间 | 高风险商业动作应保留足够审计期 |

## 4. 数据分级

| 级别 | 示例 | 控制 |
|---|---|---|
| Secret | password_hash、token 原文、Cookie、Provider Key | Token 原文不落库；密钥外置；禁止日志 |
| PII | 邮箱、联系人、电话、反馈正文 | 最小化、脱敏日志、权限控制 |
| Commercial Sensitive | 金额、范围、基线、付款、附件名 | Workspace 隔离、审计、备份 |
| Internal | 项目状态、行动项、错误码 | 授权后可见 |
| Public Minimal | 公开页必要版本内容 | 单对象公开会话授权 |

## 5. 数据保留原则

- 商业版本、决策、基线、付款、审计事件：按业务与合规策略长期保留，V0.1 不自动物理清理。
- 公开会话、验证令牌、已过期幂等记录：可按安全策略定期清理，但保留必要的哈希审计摘要。
- ActionItem 与 Feedback 投影：可重建，可按重建策略清理。
- 文件：只要存在历史 FileReference，就禁止生命周期策略物理删除。
- 日志与 IP/UA 哈希：按部署与隐私策略设置有限保留期。
