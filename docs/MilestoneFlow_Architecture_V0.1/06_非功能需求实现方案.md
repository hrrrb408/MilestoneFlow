# 《MilestoneFlow Pilot MVP V0.1 非功能需求实现方案》

## 1. 文档信息

| 字段 | 内容 |
|---|---|
| 文档编号 | MF-NFR-001 |
| 版本 | V0.1 |
| 覆盖 | 性能、可用性、可靠性、安全、可维护性、可观测性、备份恢复、兼容性和质量门禁 |

## 2. 非功能目标

| 维度 | V0.1 目标 |
|---|---|
| 核心 API 性能 | 除文件/邮件外 P95 < 500ms |
| 公开确认同步结果 | < 2s |
| 可用性 | 99.5%，计划维护除外 |
| 数据正确性 | 重复确认/付款导致重复商业结果为 0 |
| 租户安全 | 跨工作空间成功访问为 0 |
| 公开安全 | 跨公开对象和未授权文件下载为 0 |
| 数据保护 | 数据丢失为 0，备份恢复测试通过 |
| 日志安全 | 完整令牌、密码、密钥出现数为 0 |
| 客户端 | 管理端桌面优先，公开页移动端优先 |

## 3. 性能方案

### 3.1 性能预算

核心 API 的 500ms P95 预算建议：

| 环节 | 预算 |
|---|---:|
| Reverse Proxy/TLS | 20ms |
| 应用过滤器与认证 | 40ms |
| 业务逻辑 | 80ms |
| 数据库查询/事务 | 250ms |
| 序列化与网络余量 | 110ms |

该预算用于定位，不是每个请求的硬分配。

### 3.2 数据库优化

- 所有业务查询首列过滤 `workspace_id`。
- 为列表常用组合建立复合索引，例如：
  - `(workspace_id, status, updated_at desc)`；
  - `(workspace_id, project_id, created_at)`；
  - `(workspace_id, due_date, status)`；
  - `(token_hash)` 唯一索引；
  - `(status, next_attempt_at)` Worker 索引。
- Dashboard 使用只读 Projection/聚合 SQL，避免逐模块 N+1。
- JPA 默认关闭 Open Session in View。
- 所有集合读取使用分页；禁止无界审计列表。
- 慢查询阈值初始 300ms，记录 SQL 指纹而非敏感参数。

### 3.3 缓存

V0.1 不以缓存保证正确性：

- 可缓存时区、币种元数据、公开静态配置；
- 不缓存最终权限判断、支付余额、基线或幂等结果作为唯一来源；
- 本地缓存必须有短 TTL，实例重启不影响业务正确性；
- 多实例前评审 Redis。

### 3.4 前端性能

- 管理端和公开端路由分包；
- 公开页不加载后台图表和组件库全集；
- 静态资源内容哈希与长期缓存；
- HTML 和公开业务响应 `no-store`；
- 首屏关键 CSS/字体最小化；
- 图片和交付文件不经前端应用服务器转发大字节流。

## 4. 可用性与可靠性

### 4.1 事务可靠性

核心商业事务使用 PostgreSQL ACID 事务。必须测试：

- 中间步骤失败整体回滚；
- 并发动作仅一个成功；
- 重试返回同一幂等结果；
- 审计和业务状态一致；
- 持久化事件与业务事务同提交。

### 4.2 异步任务可靠性

- 邮件和非关键投影通过数据库持久化队列处理。
- 任务状态：`PENDING → PROCESSING → COMPLETED/RETRY/FAILED`。
- 使用指数退避和随机抖动。
- 临时失败可重试，永久失败停止重试。
- 每项任务有最大尝试次数和 `last_error_code`。
- Worker 崩溃后通过租约超时恢复。

### 4.3 幂等与去重

- API 幂等表防止网络重试重复副作用。
- 数据库唯一约束防止应用缺陷突破。
- 事件消费者保存处理记录或使用天然唯一业务键。
- 定时任务按业务日期和来源唯一键生成，重复扫描不重复创建逾期事件。

### 4.4 优雅降级

| 依赖故障 | 降级行为 |
|---|---|
| 邮件失败 | 发布成功，显示失败与复制链接 |
| 对象存储上传失败 | 文件不进入 AVAILABLE，阻止正式提交 |
| 对象存储下载失败 | 显示可重试，不改变验收状态 |
| 监控后端失败 | 应用继续服务，本地 stdout 日志保留 |
| 分析埋点失败 | 不影响业务事务 |
| 数据库失败 | 快速失败，不返回伪成功 |

## 5. 安全实现

### 5.1 应用安全

- Spring Security 默认拒绝未配置路由。
- 内部 Cookie 认证启用 CSRF。
- 生产同源，CORS 最小化。
- 输出编码、CSP 和框架默认模板转义防 XSS。
- ORM 参数绑定防 SQL 注入；动态排序使用字段白名单。
- 上传校验扩展名、MIME、大小、数量和对象实际元数据。
- 高熵令牌、数据库存哈希、日志脱敏。
- 公开页面不接入捕获完整 URL 的第三方分析脚本。

### 5.2 供应链安全

- 锁定依赖和构建工具版本。
- CI 执行依赖漏洞扫描、容器镜像扫描和 SBOM 生成。
- 高危可利用漏洞阻断发布。
- 基础镜像使用最小 JRE、非 root 用户和只读文件系统优先。
- 每月安排依赖升级窗口；安全 Patch 可走快速变更流程。

### 5.3 数据保护

- 全链路 HTTPS。
- 数据库和对象存储开启静态加密。
- 最小化收集客户数据，不保存不必要的合同全文和机密。
- 日志中邮箱可部分脱敏；IP 可哈希或按合规策略缩短保留。
- 生产数据访问有最小权限和操作审计。

## 6. 可观测性

### 6.1 三类信号

| 信号 | 实现 |
|---|---|
| Logs | JSON 结构化日志，stdout 输出 |
| Metrics | Micrometer，导出 Prometheus/OTLP |
| Traces | OpenTelemetry，HTTP/DB/Worker 跨进程关联 |

### 6.2 日志字段

```json
{
  "timestamp": "...",
  "level": "INFO",
  "service": "milestoneflow-api",
  "environment": "prod",
  "requestId": "...",
  "traceId": "...",
  "workspaceId": "...",
  "userId": "...",
  "publicLinkId": "...",
  "module": "quotation",
  "event": "quote_published",
  "result": "success",
  "durationMs": 123
}
```

业务正文、客户留言、完整邮箱、Token 和文件名默认不进入日志。

### 6.3 核心指标

**技术指标：**

- `http.server.requests`；
- DB 连接池、事务回滚、慢查询；
- Worker 队列长度、任务年龄和失败；
- 文件上传完成率；
- 邮件成功率；
- 限流次数；
- 容器重启、CPU、内存。

**业务正确性指标：**

- 报价发布/确认；
- 交付提交/验收；
- 变更确认；
- 应收激活；
- 付款记录；
- 核心闭环完成；
- 幂等重放次数；
- 状态冲突次数。

业务指标不得包含完整公开令牌、文件内容或客户敏感全文。

### 6.4 Trace

必须覆盖：

- API 请求 → 数据库事务；
- 发布 → 持久化事件；
- Worker 领取 → 邮件 Provider；
- 文件签名请求 → 对象存储；
- 公开链接交换 → 业务动作。

使用 `request_id` 和 `trace_id` 关联用户错误页、日志和审计。

## 7. 配置管理

### 7.1 配置分类

| 类别 | 示例 | 管理方式 |
|---|---|---|
| 非敏感静态配置 | 分页大小、上传上限 | 版本化配置文件 |
| 环境配置 | 域名、Profile、日志级别 | 环境变量 |
| 密钥 | DB 密码、Provider Key、Cookie Key | Secret Manager/部署密钥 |
| 业务可调配置 | 链接默认有效期、提醒冷却 | 配置表 + 审计 |

### 7.2 配置约束

- 使用类型安全配置绑定和启动校验。
- 默认值必须安全，例如生产不允许 debug、宽松 CORS、公开 Actuator。
- 配置变更记录操作者、前后值和时间。
- 热更新仅用于低风险参数；认证、密钥和数据库配置通过发布变更。

## 8. 可维护性

### 8.1 代码质量

- 模块边界自动测试；
- 统一格式化和静态分析；
- Domain 不依赖 Web DTO；
- Repository 不跨模块；
- 公共 API 有 Javadoc/文档；
- ADR 记录重大选择。

### 8.2 测试金字塔

| 层级 | 重点 |
|---|---|
| Unit | Money、时区、状态机、校验、令牌哈希 |
| Module Integration | 单模块 Repository、事务、事件 |
| Contract/API | OpenAPI、错误码、鉴权、幂等 |
| E2E | 8 条重点 Pilot 链路 |
| Security | 租户越权、公开对象越权、文件授权、CSRF |
| Performance | Dashboard、公开确认、付款并发 |

### 8.3 数据迁移质量

- 每个迁移在空库和上一生产快照恢复库验证。
- 迁移脚本不可修改已发布版本。
- 大数据回填使用可重入批处理。
- 迁移后执行约束、索引和关键计数校验。

## 9. 可访问性与兼容性

### 9.1 浏览器

支持当前主流桌面浏览器最近两个稳定大版本；公开页同时测试 iOS Safari 和 Android Chrome 的常见版本。具体版本矩阵由测试窗口在执行时冻结。

### 9.2 响应式

- 管理端最低推荐 1280px；关键表单在 1024px 可用。
- 公开页最低 360px，无横向滚动完成确认。
- 触控目标、键盘导航、焦点状态和错误提示可感知。

### 9.3 可访问性

目标对齐 WCAG 2.1 AA 的核心要求：

- 表单 Label/错误关联；
- 颜色不是唯一状态表达；
- 可键盘操作；
- 焦点顺序合理；
- 对话框焦点管理；
- 文件和金额信息可被屏幕阅读器理解。

## 10. 容量与扩展

### 10.1 Pilot 假设

架构按以下量级验证，不作为商业上限：

- 100 个活跃工作空间；
- 每工作空间 1,000 个项目以内；
- 每项目几十个版本/事件；
- 文件总量主要由对象存储承担；
- 峰值并发低于 100 请求/秒。

### 10.2 扩容触发

| 信号 | 动作 |
|---|---|
| API CPU 持续 > 70% | 增加 API 实例 |
| Worker 积压增长 | 增加 Worker 或分队列 |
| DB CPU/IO 成瓶颈 | 索引、查询优化、提升规格、读副本评审 |
| 分布式限流需求 | 引入 Redis |
| 模块独立发布成为团队瓶颈 | 评估服务拆分 |
| 搜索需求复杂且数据量大 | 评估搜索引擎，不提前引入 |

## 11. 备份恢复与数据一致性

- 数据库自动备份 + PITR。
- 对象存储版本控制。
- 每月恢复演练。
- 恢复后执行业务对账：版本数、基线链、应收余额、文件引用、审计连续性。
- 审计事件不是数据库备份的替代品，但可用于确认恢复结果。

## 12. SLO 与错误预算

99.5% 月可用性约允许 3 小时 39 分钟不可用。错误预算消耗超过 50% 时冻结非必要功能发布，优先处理稳定性；超过 100% 时只允许修复、安全和恢复类变更。

## 13. 发布质量门禁

- 22 项 P0 需求均有架构/接口/测试映射。
- 重点 E2E 全部通过。
- 无 P0/P1 阻断缺陷。
- 安全测试中越权成功数为 0。
- 并发重复业务副作用为 0。
- P95 和公开确认目标通过基准测试。
- 备份恢复、迁移和回滚演练通过。
- 生产配置安全检查通过。
- 依赖和镜像无未接受的高危漏洞。
